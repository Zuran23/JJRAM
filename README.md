Branch Flow

1. Work goes to feature branches, then open a PR into testing-branch.
2. Only PRs from testing-branch can go into Approved-branch.
3. Only PRs from Approved-branch can go into master.

Required GitHub Settings

1. Protect testing-branch, Approved-branch, and master.
2. Require pull requests before merging.
3. Require the CI workflow to pass before merging.
4. Disable direct pushes to Approved-branch and master.
